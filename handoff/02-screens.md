# 02 — 画面仕様（9 枚）

`<VariantCv2 initialScreen="..."/>` で各画面を確認できます。番号は推奨実装順。

---

## 1. ランキング（`initialScreen="home"`）

**役割**: トップ画面。挑戦中プロジェクトを一覧、過去プロジェクトもライブラリで保持。

**レイアウト**: 左 1.7fr × 右 1fr
- 左: アクティブプロジェクトカード (2 列 × 3 行) → 進捗ランキング順 / 過去プロジェクトライブラリ (一覧)
- 右: 「テーマ出題のポイント」3 条件カード / 今週のクエスト (glass-dark)

**プロジェクトカード（1枚 ≒ 380×100）**
- 左: `<RingV2 size={56} value={progress}/>` + 上位3位なら右上にメダル小バッジ
- 右: チーム名 / プロジェクト名 / 連続日数 / タスク完了率 / 次マイルストーン
- バッジは `data-c-fun="playful"` で囲み、Pro モードで非表示

**プロジェクトライブラリ（圏外）**
- ステータス: `active / paused / completed / archived`
- 各 1 行に: ●ステータス丸 / プロジェクト名 + メモ / ●ステータスチップ / アクションボタン (再開/振り返り/復元)

**フィルターセグメント**: 進行中 / 休止 / 完了 / すべて

**今週のクエスト**: 3 タップ進捗バー + 達成条件 + 残り日数 (`data-c-fun="playful"`)

---

## 2. ダッシュボード（`initialScreen="dash"`）

**役割**: プロジェクトひとつの今日の状態。

**レイアウト**: タテに重なるセクション
1. ヘッダー: プロジェクトアイコン + 名前 + バッジチップ (`data-c-fun="playful"`)
2. メトリクス 4 枚 (1×4 grid): 各 `<RingV2 size={66}/>` + ラベル + 副情報 + チップ
3. 左 1.5fr マイルストーンバー (横並び 6 ポイント) + 進行中タスクリスト / 右 1fr バッジコレクション + 直近イベント

**マイルストーン横バー**
- 6 マイルストーン点 → 完了は黒丸 / 進行中は accent (`var(--c-accent)`) + アクセントソフトのリング
- 進捗線は 0% → 52% の位置まで黒→accent グラデ

**進行中タスクリスト**
- 5 列 grid: チェック / タイトル / 担当 / 期限 / タグ
- ステータス円の色: done=ok / review=warn / doing=accent / todo=mute

**コンフェッティ** (`data-c-fun="playful"`): 画面マウント時 24 個の小片が降る (1.4s 一過性)

---

## 3. 実行計画（`initialScreen="plan"`）

**役割**: Why / Who / What / How を一覧編集 + 4P + KPI。診断は別画面（`diag`）に分離。

**レイアウト**: 左 1.2fr × 右 1fr
- 左: 4 つの Why/Who/What/How カード（縦並び）+ `glass-dark` の AI ヒント
- 右: 4P マーケティングミックスカード / 🎯 目標カード（定性 + 定量 KPI 3 つ）

**Why/Who/What/How カード**
- 左: 黒タイル (48×48) に "Why" 等の文字 (角丸 14)
- 中: 絵文字 + フレームワーク説明 + ホバーで `?` ツールチップ + 本文
- 右: `<RingV2 size={48} value={score}/>` (AI 評価 0–100)

**4P カード**: 4 つを 2×2 grid。各タイル = 絵文字 + 名前 + 説明文

**🎯 目標カード**:
- 定性ブロック (柔らかい長文)
- 定量 KPI 3 行: ラベル + プログレスバー (`var(--c-accent)`) + 値

## 4. WBS / ガント（`initialScreen="wbs"`）

**役割**: 全タスクの階層・期間・進捗を 1 画面で可視化。

**レイアウト**:
- ヘッダー: ビュー切替 (ガント / ツリー / カンバン) + 期間 + ＋新規
- 統計 3 枚 (完了 / 進行中 / 期限超過)
- メイン: ガント (左: タスクツリー 320px / 右: 週軸)

**ガント仕様**
- 列幅: 1 週 = 24 px (期間 9月〜3月 = 28 週)
- フェーズ行: トグル ▸/▾、太字、進捗バー
- タスク行: インデント 28px、左にタスク名 / 担当 / 進捗%、右に進捗バー
- マイルストーン: 14×14 菱形 (`accent`、白枠 2px)
- Today: 縦線 (`accent`、2px) + "Today" ピル

**カラー**:
- 完了フェーズ → `var(--ok)`
- 進行中フェーズ → `var(--c-accent)` (currentフラグあるなら太く)
- 子タスクバー → グレー `rgba(150,170,200,.5)` を背景、進捗だけ accent

## 5. 収支計画（`initialScreen="budget"`）

**役割**: NEO 基金 + 協賛 + 自己資金 と 支出計画 vs 実績。

**レイアウト**:
- 上: サマリー 4 枚 (収入計画 / 収入実績 / 支出実績 / 差引利益)
- 下: 左 1.5fr テーブル / 右 1fr 月次推移チャート + AI 観察カード

**サマリーカード**:
- 大数値 (JetBrains Mono, 22px, weight 800) + 副情報チップ

**テーブル**:
- 列: 項目 / 計画 / 実績 / 差異 / %
- 収入グループは緑系 `rgba(180,220,200,.25)` 背景、支出グループはオレンジ系 `rgba(220,200,200,.25)`
- 行: 計画 (mute) / 実績 (太字) / 差異 (green/orange) / % (green if ≥80, orange if ≥50)
- "未確定" チップ (accent-soft)

**月次推移**:
- 100×130 SVG、月ごとに収入(accent) + 支出(warn) のペアバー
- 将来月は opacity 0.5

**AI 観察カード** (`glass-tint` あるいは accent border):
- 「✦ NEO.ai からの観察」+ 具体的なアラート + アクションボタン 2 つ

## 6. 診断レポート（`initialScreen="diag"`）

**役割**: 14 項目の評価を週次でトラッキング + AI コメント。

**レイアウト**:
- 上: 左 1.1fr 大型ヘックスレーダー + スコア / 右 1fr 週次推移チャート
- 下: 14 行のドリルダウンテーブル

**ヘックスレーダー** (`<HexRadar size={260}/>`):
- 14 軸を等間隔配置、3 段のレベル線
- 現在値の多角形は accent (fill rgba(91,141,239,.12), stroke `var(--c-accent)`)
- 各軸の頂点に黒ドット (4px) + 白枠

**スコア表示** (右):
- 大数値 "27 / 42" (JetBrains Mono, 46px)
- ティアバッジ "★★★ ベテラン伴走" 等
- 強み 8 / 注意 5 / 要支援 1 のサマリ
- `glass` インナーボックスで総評（4 週で +6）

**週次推移チャート**:
- 240×130 SVG、5 点ポリライン (4週前→今週)
- 各点に値ラベル、accent ライン (2.5px)

**ドリルダウンテーブル**:
- 列: 項目 / 評価 (○△×) / 推移 (sparkline + delta) / AI コメント / 推奨アクション
- 各 sparkline: 60×18 SVG、accent ライン + 最終点に丸

## 7. NEO 基金申請（`initialScreen="fund"`）

**役割**: 申請ステータス追跡 + 用途内訳 + AI 添削。

**レイアウト**:
- ステータスパイプライン (4 ステップ横並び)
- 左 1.4fr: 申請内容フォーム + 用途内訳テーブル
- 右 1fr: AI 添削カード (`glass-dark`) + 過去申請履歴

**ステータスパイプライン**:
- 4 円 (下書き → 一次審査 → 二次審査 → 承認・送金)
- 完了円は黒、現在円は `accent` + ソフトリング (5px)
- 円同士を 2px 横線で接続 (完了部分は黒)

**申請フォーム**:
- 申請額: 大数値 (22px) + 副バッジ "中間報告 加算分"
- プロジェクト名 / 申請理由 (textarea readOnly) / 添付ファイルチップ + ＋追加ボタン

**用途内訳テーブル**:
- 項目 / プログレスバー (gradient accent → accent-deep) / 金額
- 合計行は太い罫線 + 太字

**AI 添削カード** (`glass-dark`):
- 3 つの改善ポイント、番号丸 (warn/ok) + 内容 + 副情報 + › アイコン
- 「✦ 改善案をまとめて適用」ボタン

**過去申請履歴**: 4 列 grid (名前 / 金額 / 日付 / ステータス)

## 8. AI 伴走者（`initialScreen="ai"`）

**役割**: フルスクリーンの AI 対話画面（浮遊パネルの大画面版）。

**レイアウト**: 左 1.2fr × 右 1fr
- 左: チャット履歴 + 入力欄
- 右: 提案カードスタック

**チャット**:
- AI バブル: 白 (border line) + AI 円アバター (黒 + ✦)
- ユーザーバブル: 黒 + ユーザー円アバター (accent グラデ)
- アバター 30×30、角丸 10、入っているのは絵文字 1 文字

**入力欄**: ボトムスティック、`glass` 表面 + 送信ボタン (黒)

**提案カード（右）**:
- 各カード: kind チップ (黒) + 内容 + ✓承認 / 却下 / ↻ 別案 ボタン群
- 承認済みは緑ボーダー + チップ「✓ 反映済み」
- 「AI に任せる」トグル

## 9. テーマ出題（`initialScreen="theme"`）

**役割**: 企業 PR 担当が出題テーマを作成・管理。

**レイアウト**: 左 1.2fr × 右 1fr
- 上: ステータスチップ群 (公開中3 / 下書き1 / 終了1 / 計5) + ＋新規テーマ
- 左下: テーマ一覧 (3 枚 + 残りリスト)
- 右下: 📝 テーマ詳細 + 🌐 公開プレビュー

**テーマカード**:
- code (JetBrains Mono) + ステータスチップ + カテゴリ
- タイトル (14px, 太字) + 主催 + 締切 / 応募数 / 採択数

**公開プレビュー**:
- `glass-dark` の大型カード
- code + 主催 + タイトル (大) + 説明文 + 締切&特典の 2 列 + 「同意して応募 →」ボタン
